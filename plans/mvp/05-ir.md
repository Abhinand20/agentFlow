# M5 - IR

- Milestone: M5
- Version: v0.1 (MVP) — Language Level A
- Status: Planned
- Spec: [§4 Execution/data model](../../spec/grammar.md#4-execution-and-data-model), [§8 Model resolution](../../spec/grammar.md#8-model-and-provider-resolution), [§13.2 IR](../../spec/grammar.md#132-ir-and-bindings)

## Goal

Produce a normalized, binding-agnostic IR with deterministic JSON. The IR is the
stable contract for render (M6) and bindings (M7/M10); AST/model are never
consumed downstream.

## Scope

### In scope
- `ir.Program` from resolved flow + model, including data-model metadata.
- Deterministic JSON serialization.

### Out of scope (deferred)
- Linear Plan IR (M14).

## Packages & files

- `internal/ir/ir.go`
- `internal/ir/json.go`
- `internal/ir/from_resolved.go`

## Tasks

- `ir.Program` fields:
  - **Entry** — trigger (`on:`), flow name, input type.
  - **Agents** — resolved `(provider, alias)`, host model id slot, `OutEnum`,
    `Retry`, tools, prompt, permissions.
  - **Capabilities** — MCP + model-provider blocks.
  - **Gates** — script path, `OnFail` action + target control label, behavior, script retry.
  - **Flow** — resolved tree with control/value labels, sequential edges, gather payloads,
    branch heads (value label + enum), loop bounds, return bindings.
- `ir.FromResolved(model, resolved) (Program, Diagnostics)`.
- `MarshalJSON` with sorted keys for snapshot stability.
- Debug dump: `af` can print IR (wired in M8).

## Data shapes / snippets

```go
type Agent struct {
    Name, Prompt, Permissions string
    Provider, Alias, HostModelID string
    In, Out string
    OutEnum []string
    Retry   int
}
type Gate struct {
    Name, Run string
    OnFail       GateFailAction
    OnFailTarget string // control label
    Behavior     string
}
type GatherPayload struct {
    GatherStep string
    Branches   map[string]string // branch control label -> producing value label
}
```

## Acceptance criteria

- IR JSON snapshot for [examples/review.af](../../examples/review.af).
- Gate `quality` IR shows `OnFail: retry`, `OnFailTarget: build`.
- `code_review` IR shows `Return: review`, gather payload on `review`.
- Serializing twice yields identical bytes.

## Dependencies

- M2 (model), M3 (resolved flow).

## Risks / notes

- Ambiguous bare model aliases must not reach IR (`AF110` in M2).
- `OutEnum` drives both validation (M4) and output protocol (M6).
