## 1. Goal

Produce a normalized, **binding-agnostic** IR with deterministic JSON serialization. The
IR is the **stable contract** consumed by render (M6) and bindings (M7/M10). After M5,
nothing downstream touches `ast`, `model`, or `flowgraph` — they consume `ir.Program`
only. This isolation is what lets multiple hosts (Claude, Cursor, SDK) share one front
end.

The existing `internal/ir/ir.go` is the M0 empty stub (`type Program struct{}`). M5
replaces it with the real shape.

## 2. Deliverables

- `internal/ir/ir.go` — `Program` and all node types (replaces stub).
- `internal/ir/from_resolved.go` — `FromResolved(prog *model.Program, res *flowgraph.Resolved) (Program, diag.Diagnostics)`.
- `internal/ir/json.go` — deterministic `MarshalJSON` (sorted keys) + `Marshal(p) []byte`.
- IR JSON snapshot goldens in `internal/ir/testdata/`.

## 3. IR shape (`internal/ir/ir.go`)

The IR is a flattened, fully-resolved description. It carries **no positions** (it is a
build artifact, not a diagnostic source) — this also makes JSON snapshots robust.

```go
package ir

type Program struct {
    Entry        Entry
    Agents       []Agent       // sorted by Name
    Gates        []Gate        // sorted by Name
    Capabilities []Capability  // sorted by Name
    Flow         Flow          // the single inlined entry flow tree
}

type Entry struct {
    Trigger  string // on: value, e.g. "/ship"
    FlowName string
    InType   string // input type (opaque nominal ok), "" if none
    OutType  string // resolved output type
}

type Agent struct {
    Name        string
    Provider    string // resolved model provider
    Alias       string // resolved symbolic alias
    HostModelID string // EMPTY in IR — bindings fill this (see §3.1)
    In          string
    Out         string   // enum type name or "text" or ""
    OutEnum     []string // enum members if Out is an enum
    Tools       []ToolRef
    Prompt      string // resolved base prompt text (NO output-protocol suffix)
    Permissions string
    Retry       int    // output-protocol parse retries
    Description string
}

type ToolRef struct{ Capability, Tool string }

type Gate struct {
    Name         string
    Run          string
    OnFail       string // "halt" | "retry" | "goto" | "enter-loop" (string for JSON stability)
    OnFailTarget string // control label
    Behavior     string // "blocking" | "advisory"
    ScriptRetry  int
}

type Capability struct {
    Name      string
    Kind      string   // "model-provider" | "mcp"
    Models    []string
    Tools     []string
    Transport string
    Command   string
    Args      []string
}

// Flow is the inlined kernel tree (mirrors flowgraph.Node, but position-free
// and self-contained for downstream consumers).
type Flow struct {
    Name    string
    Return  ReturnBinding
    Root    Node
    // Linear control-label order for runbook numbering.
    Order   []string
}

type ReturnBinding struct {
    ValueLabel     string
    OutType        string
    Defaulted      bool
    BranchTerminal bool
}

type NodeKind string
const (
    NodeSeq      NodeKind = "seq"
    NodeStep     NodeKind = "step"
    NodeBranch   NodeKind = "branch"
    NodeLoop     NodeKind = "loop"
    NodeParallel NodeKind = "parallel"
)

type Node struct {
    Kind NodeKind

    // seq
    Steps []Node `json:",omitempty"`

    // step (leaf)
    Step *StepRef `json:",omitempty"`

    // branch
    BranchValue string        `json:",omitempty"`
    BranchEnum  []string      `json:",omitempty"`
    Cases       []BranchCase  `json:",omitempty"`

    // loop / repeat
    DoWhile  bool   `json:",omitempty"`
    Cond     *Cond  `json:",omitempty"`
    Max      int    `json:",omitempty"`
    HasMax   bool   `json:",omitempty"`
    Body     *Node  `json:",omitempty"`

    // parallel
    Branches []Node         `json:",omitempty"`
    Gather   *StepRef       `json:",omitempty"`
    Payload  *GatherPayload `json:",omitempty"`
}

type StepRef struct {
    ControlLabel string
    ValueLabel   string // "" if no output
    Decl         string // agent/gate/terminal name
    Kind         string // "agent" | "gate" | "terminal"
    OutType      string
    OutEnum      []string
    // PrevProducer is the control label whose latest output feeds this step in
    // sequence (spec §4.5); "" if none. Precomputed so render needn't walk the tree.
    PrevProducer string
}

type BranchCase struct{ Values []string; Body Node }
type Cond struct{ ValueLabel, Op, Enum string }
type GatherPayload struct {
    GatherControlLabel string
    Branches           []GatherBranch
}
type GatherBranch struct{ ControlLabel, ValueLabel string }
```

### 3.1 `HostModelID` is intentionally empty in IR

Model alias → host model id is a **binding** concern (spec §8: "Bindings map resolved
`(provider, alias)` to host-native model id strings"). The IR carries `(Provider, Alias)`
only; M7's Claude binding fills `HostModelID`. This keeps the IR host-neutral. Document
this clearly so M7 does not expect it pre-filled.

## 4. `FromResolved` (`internal/ir/from_resolved.go`)

```go
func FromResolved(prog *model.Program, res *flowgraph.Resolved) (Program, diag.Diagnostics)
```

Pure translation; assumes a **valid** program (M4 passed). It may still return
diagnostics for invariant violations it discovers (defensive), but in the happy path
returns none. Steps:

1. **Agents** — for each agent referenced by the resolved tree (and, for completeness,
   all declared agents — decide: emit only *reachable* agents to keep output minimal;
   **decision: emit only agents reachable from the entry flow**, matching what bindings
   need; unreachable ones already warned via `AF207`). Copy resolved `(Provider, Alias)`,
   `Out`/`OutEnum` (look up enum members from `prog.Types`), tools, `Prompt` (base text),
   permissions, retry.
2. **Gates** — reachable gates; map `GateFailAction` enum → string.
3. **Capabilities** — all `model-provider` and `mcp` capabilities actually used
   (providers backing reachable agents; mcp capabilities backing referenced tools). Decide
   whether to emit unused capabilities; **decision: emit only used capabilities** for a
   tight `.mcp.json`/settings later.
4. **Flow** — translate `flowgraph.Node` → `ir.Node` recursively; attach `OutEnum` to
   branch/step nodes by resolving the value label's type via M3 metadata; compute
   `PrevProducer` per step (reuse M3's `PrevProducer` helper); copy `ReturnBinding`.
5. **Entry** — trigger, flow name, in/out types (out resolved from the return binding when
   not explicit).
6. **Sort** all top-level slices by name; preserve tree/`Order`/payload ordering.

### 4.1 Enum attachment

`OutEnum` appears in three places (agent, branch, step). Source of truth is
`prog.Types[enumName].Values`. Centralize a helper `enumMembers(prog, typeName) []string`
that returns `nil` for `text`/unknown. Attach consistently so render/binding never look up
types themselves.

## 5. Deterministic JSON (`internal/ir/json.go`)

Go's `encoding/json` already emits **struct fields in declaration order** and sorts **map
keys**. Since the IR uses slices (not maps) for all collections and we pre-sort them,
standard `json.MarshalIndent(p, "", "  ")` is deterministic **provided** we avoid maps in
the IR (we do). Therefore:

```go
func Marshal(p Program) ([]byte, error) {
    return json.MarshalIndent(p, "", "  ")
}
```

- No custom `MarshalJSON` is strictly required if the IR contains no maps. **Decision:
  keep the IR map-free** (use ordered slices everywhere) so we rely on stdlib determinism
  rather than hand-rolled marshaling. If a map is unavoidable later, add a custom
  marshaler then.
- `omitempty` on optional `Node` fields keeps the JSON compact and stable across node
  kinds. Verify that `omitempty` does not hide a meaningful zero (e.g. `Max: 0` with
  `HasMax: true` — guard by always emitting `HasMax`; do **not** `omitempty` `Kind`).
  Re-examine each `omitempty` against this hazard.

> **Idempotence test:** marshal twice, assert byte-identical (acceptance criterion).

### 7.1 Snapshot goldens

  - `reviewer` agent `OutEnum: ["approve","revise","reject"]`, `Prompt` = base reviewer
    text **without** the `agentflow-output` suffix.
  - `ship` flow `Return.BranchTerminal: true`; branch node `BranchEnum` = Verdict members.
- `pipeline.af`: `Return.Defaulted: true`, `ValueLabel: "edit"`, `OutType: "text"`.
- `critic.af`: loop node `DoWhile: true`, `Cond{verdict == pass}`, `Max: 3`.
- `research.af`: parallel/gather payload with three branches.

### 7.2 Property tests

- Marshal twice → identical bytes (idempotence).
- No Go `map` types in the IR (a reflection-based test that walks `Program` and fails on
  any `reflect.Map` keeps the determinism guarantee enforced).
- Round-trip: `Marshal` → `json.Unmarshal` into `Program` → `Marshal` again → identical
  (proves the JSON fully captures the IR; catches missing exported fields).

## 10. Risks & notes

- **`omitempty` zero-value traps.** The most likely source of a subtle IR bug. Audit every
  `omitempty` field; never `omitempty` a discriminant (`Kind`) or a flag whose `false` is
  meaningful alongside another field (`HasMax`/`Max`). Prefer explicit booleans over
  relying on absence.
- **Reachable-only emission.** Emitting only reachable agents/gates/capabilities keeps the
  IR (and downstream files) minimal, but means an unreachable agent never reaches a
  binding. This is intended (it is already `AF207`-warned). Document so M7 authors do not
  expect every declared agent.
- **Stub replacement.** Replacing `type Program struct{}` will break the M0 binding test's
  `Emit(ir.Program)` signature only if it referenced fields (it does not — it passes an
  empty `Program{}`). Confirm `internal/binding/binding_test.go` still compiles; the
  `Emit(_ ir.Program)` stub takes the type by value and is unaffected.
- The IR is the **stable contract**: once M7 ships, changing IR field names is a regold
  across render + binding goldens. Treat additions as cheap, renames/removals as breaking.
