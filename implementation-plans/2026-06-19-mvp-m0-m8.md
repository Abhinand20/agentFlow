## Diagnostic codes introduced by the execution plans

The execution plans refine the catalog in [plans/README.md](../plans/README.md) with
several codes the high-level specs did not enumerate. **When implementing, fold these into
the authoritative catalog** (plans/README.md and plans/mvp/04-validation.md):

| Code | Sev | Milestone | Meaning |
|------|-----|-----------|---------|
| `AF120` | warn | M2 | Unknown field (preserved in `Raw`) |
| `AF130` | error | M2 | mcp capability missing `command` |
| `AF131` | error | M2 | model-provider missing `models` |
| `AF132` | error | M2 | agent missing `model` |
| `AF133` | warn | M2 | duplicate enum member |
| `AF134` | error | M2 | invalid `on-fail` (incl. `bounce-back`) |
| `AF135` | error | M2 | `on-fail: retry`/`goto` missing `on-fail-target` |
| `AF136` | warn | M2 | `on-fail-target` present for `halt`/`enter-loop` |
| `AF137` | warn | M2 | `on:` on a non-entry flow |
| `AF138` | error | M2 | no entry flow |
| `AF139` | error | M2 | multiple entry flows |
| `AF212` | error | M3 | recursive flow nesting |
| `AF213` | error | M3 | value label written with conflicting output types |
| `AF300`–`AF303` | warn | M7 | advisory-tier capability fallbacks (parallel/loop/output/gate) |

> These are design decisions captured in the execution plans; treat the validation-rule
> table in [plans/mvp/04-validation.md](../plans/mvp/04-validation.md) as the place to
> reconcile the final, authoritative list before/while implementing M2–M4.
