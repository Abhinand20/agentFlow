## 3. Shared pipeline (`internal/pipeline/pipeline.go`)

```go
// Compile runs parse -> resolve -> inline/normalize -> validate -> IR.
// It returns the IR (zero value if errors) and ALL accumulated diagnostics.
func Compile(path string) (ir.Program, diag.Diagnostics) {
    src, err := os.ReadFile(path)
    if err != nil { /* return a single AF000-ish read error diag */ }

    var diags diag.Diagnostics
    astRoot, d := parser.Parse(path, string(src)); diags.Add(d...)
    if diags.HasErrors() { return ir.Program{}, diags }

    srcDir := filepath.Dir(path)
    prog, d := sema.Resolve(astRoot, srcDir); diags.Add(d...)
    if diags.HasErrors() { return ir.Program{}, diags }

    res, d := flowgraph.Resolve(prog); diags.Add(d...)   // emits AF212/AF213
    // Continue to validate even with M3 errors? Decision: if M3 produced a partial
    // tree (recursion cut), still run validate to surface more issues, but do NOT
    // build IR if HasErrors.
    vd := sema.Validate(prog, res); diags.Add(vd...)
    if diags.HasErrors() { return ir.Program{}, diags }

    program, d := ir.FromResolved(prog, res); diags.Add(d...)
    return program, diags
}
```

- **Single entry point** for CLI, E2E, and future LSP (M16) / simulator (M14). Do not
  duplicate this sequence anywhere else.
- **Diagnostic accumulation:** gather warnings even when there are no errors (so `af
  validate` can print warnings and still exit 0).
- **Source caching:** the renderer in M6 needs no source, but diagnostics rendering
  (`diag.Render`) needs the original source string. Either return the source from `Compile`
  or have the CLI re-read it. **Decision:** add `CompileResult{ IR, Diags, Source string }`
  or a sibling `CompileSource(path) (string, ...)`; simplest is for the CLI to read the
  file once and pass both path and source. Pick one and keep it consistent.

> Recommended signature refinement:
> ```go
> type Result struct { IR ir.Program; Source string }
> func Compile(path string) (Result, diag.Diagnostics)
> ```
> so the CLI can render diagnostics against `Result.Source` without a second read.

## 4. CLI commands (`cmd/af/`)

Replace the M0 stubs. Keep the existing usage banner/structure; flesh out each command.

### 4.1 `af validate <file>`

- Run `pipeline.Compile`.
- Print `diag.Render(source, diags)` to stderr.
- **Exit policy:** `diags.HasErrors()` → exit 1; warnings only → exit 0 (print warnings).
  Empty/clean → print nothing (or a terse "ok") and exit 0. **Decision: print "ok" on
  clean** for friendliness; keep machine-parseable diagnostics on stderr.

### 4.2 `af graph <file> [--target claude-code|cursor]`

- `pipeline.Compile`; if errors, print + exit 1.
- `dot.Emit(ir)` → stdout. DOT is target-neutral; flag accepted for forward compatibility.
  **Note:** DOT emitter not yet implemented (planned M7/M8).

### 4.3 `af build <file> --target <host> [--out dir]`

Hosts: `claude-code` (M7, planned), `cursor` (M10 lean cut, [PR #10](https://github.com/Abhinand20/agentFlow/pull/10)).

- `pipeline.Compile`; if errors, print + exit 1. Print warnings (incl. `AF3xx`) to stderr.
- `b, ok := binding.Get(target)`; unknown target → error listing `binding.Names()`.
- `fs, bdiags := b.Emit(ir)`; add `bdiags` (the `AF3xx` negotiation warnings).
- `--out` defaults to `.` (current dir). `fs.Flush(out)`.
- Print a summary of written paths (`fs.Paths()`).
- Optional debug: `--emit-ir` prints `ir.Marshal(program)` to stdout instead of writing
  files (wires the M5 debug dump).

### 4.4 Flag parsing

Use the stdlib `flag` package per-subcommand (a `flag.NewFlagSet` per command), or keep the
current hand-rolled dispatch and parse `--target`/`--out` manually. **Decision: per-command
`flag.FlagSet`** for clean `-h` output. Update `usageText()` to document flags.

### 4.5 Binding import

Blank-import bindings in `cmd/af` so `init()` registration runs:

```go
import _ "github.com/Abhinand20/agentFlow/internal/binding/cursor" // M10 — done (bindings.go)
```

## 7. E2E test

`TestE2E_ReviewAf` (in `cmd/af` or `internal/pipeline`):

1. Build `review.af` into a `t.TempDir()` via `pipeline.Compile` + `claude.Emit` +
   `fs.Flush` (or by invoking the `af build` binary like the existing `cmd/af/main_test.go`
   does with `go run`).
2. Assert the file tree exists (paths from §3 of the M7 plan).
3. Assert key content goldens:
   - `.claude/commands/ship.md` contains the §9 parse instructions and the gate retry to
     control label `build`.
   - `.claude/agents/reviewer.md` contains the `agentflow-output` block.
   - `.mcp.json` contains the `github` server.
4. `af validate review.af` → exit 0, no errors.
5. `af graph review.af` → DOT containing `code_review.build` and gather edges.

Prefer driving the real binary (`go run ./cmd/af ...`) for at least one E2E path so flag
parsing and registration are exercised end to end, consistent with the M0 test style.

## 10. Risks & notes

- **Exit-code policy** must be consistent and documented (errors → 1, warnings → 0). CI and
  users depend on it.
- **Working directory for gate scripts.** The `run:` path is relative; document the assumed
  cwd at runtime (the project root where `.claude/` lives). Mismatch is a common foot-gun.
- **Regold discipline (cross-cutting practice #8).** Any spec/example drift: edit
  `review.af` first, then regenerate every stage golden. Provide a one-liner (e.g.
  `go test ./... -run Golden -update`) and record it in the README.
- **E2E via real binary vs in-process.** In-process is faster and easier to assert; one
  `go run` path is worth keeping for integration realism. Do both: in-process for content
  goldens, one binary invocation for the CLI contract.
- This is the MVP finish line; after green, update [progress/](../progress/) and flip the
  umbrella plan's status. M9+ (Level B) builds on the same `pipeline.Compile`.
