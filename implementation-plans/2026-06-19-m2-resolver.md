## 1. Goal

Lower the generic AST into a **typed semantic model** with symbol tables, by
interpreting each block's fields according to its declaration kind. This is the first
pass that assigns *meaning*. It also performs the resolutions that are local to a
single declaration: model/provider resolution (`AF110`), Level B rejection (`AF150`),
gate `on-fail` parsing, flow header interpretation, and **prompt source resolution**
(reading `prompt-file:`/`.md`-path prompts off disk into resolved text).

Cross-flow concerns — inlining, label assignment, gather payloads, return-binding
defaulting — are **not** here; they are M3. Validation rules (`AF2xx`) are M4. M2
produces just enough structure for M3 to operate and records explicit/defaulted markers
so downstream passes are deterministic.

## 2. Deliverables

- `internal/model/model.go` — typed model types + enums.
- `internal/sema/resolve.go` — `Resolve(root *ast.AST, srcDir string) (*model.Program, diag.Diagnostics)`.
- `internal/sema/fields.go` — table-driven per-kind field interpreters.
- `internal/sema/prompt.go` — prompt source resolution + path safety.
- Model snapshot goldens in `internal/sema/testdata/`.

## 3. Model types (`internal/model/model.go`)

```go
package model

import "github.com/alecthomas/participle/v2/lexer"

type Program struct {
    Capabilities map[string]*Capability // by name
    Types        map[string]*EnumType   // by name
    Agents       map[string]*Agent      // by name
    Gates        map[string]*Gate       // by name
    Flows        map[string]*Flow       // by name
    EntryFlow    string                 // name of the single entry flow ("" if none/invalid)

    // Declaration order, for deterministic iteration + snapshots.
    Order []DeclRef
}

type DeclRef struct {
    Kind DeclKind // Capability | Type | Agent | Gate | Flow
    Name string
}

type Capability struct {
    Name      string
    Kind      string   // "model-provider" | "mcp" | (other, preserved)
    Models    []string // for model-provider
    Tools     []string // for mcp
    Transport string   // for mcp (e.g. "stdio")
    Command   string   // for mcp
    Args      []string // for mcp
    Raw       map[string]ast.Value // unknown/preserved fields
    Pos       lexer.Position
}

type EnumType struct {
    Name   string
    Values []string
    Pos    lexer.Position
}

type ToolRef struct {
    Capability string // qualifier before the dot ("github")
    Tool       string // after the dot ("get_pr"); empty if unqualified
    Raw        string // original text
    Pos        lexer.Position
}

type Agent struct {
    Name          string
    ModelAlias    string // symbolic alias as written ("opus" or "anthropic.opus")
    ModelProvider string // resolved provider name (filled by model resolution)
    ResolvedAlias string // bare alias after qualifier strip ("opus")
    In            string // type name (opaque nominal ok), "" if absent
    Out           string // enum type name or "text", "" if absent
    Permissions   string
    Prompt        string // RESOLVED final prompt text (inline or file contents)
    PromptPath    string // source path if sourced from a file; "" if inline
    PromptFromFile bool
    Tools         []ToolRef
    Retry         int    // output-protocol parse retries; default 0
    Description   string
    Raw           map[string]ast.Value
    Pos           lexer.Position
}

type GateFailAction int

const (
    FailHalt GateFailAction = iota // "halt" (default when on-fail absent)
    FailRetryStep                  // "retry"
    FailGotoStep                   // "goto"
    FailEnterLoop                  // "enter-loop"
)

type Gate struct {
    Name         string
    Run          string
    OnFail       GateFailAction
    OnFailTarget string // control label, required for retry/goto
    Behavior     string // "blocking" | "advisory" (default "advisory" if absent — see §5.5)
    ScriptRetry  int    // script exit-code retries before on-fail action; default 0
    Raw          map[string]ast.Value
    Pos          lexer.Position
}

type Flow struct {
    Name   string
    Entry  bool
    On     string // trigger, e.g. "/ship"
    In     string
    Out    string
    Return string

    OutExplicit    bool // false => inferred (M3/M5 fill it)
    ReturnExplicit bool // false => default terminal producer (§4.4 Rule 0)

    Params []Param // Level B; non-empty triggers AF150
    Body   []Step  // resolved step tree (still per-flow, not inlined)
    Pos    lexer.Position
}

type Param struct{ Name, Type string; Pos lexer.Position }
```

### 3.1 Resolved step tree

M2 produces a **resolved** mirror of the AST step union: same shape, but references are
kept as names (resolution to declarations is M3's inlining concern / M4's `AF201`), and
Level B markers are surfaced. Keep it close to the AST to minimize translation bugs.

```go
type StepKind int
const (
    StepRef StepKind = iota // bare agent/gate/subflow reference
    StepChain               // a -> b -> c (sequence of refs)
    StepBranch
    StepLoop
    StepRepeat
    StepParallel
    // Level B markers (rejected at resolve, but represented for good diagnostics):
    StepCall
)

type Step struct {
    Kind  StepKind
    Pos   lexer.Position

    // StepRef / chain element
    Ref   string // declaration name as written (qual-name joined by ".")
    Alias string // "as <alias>" value label, "" if none

    // StepChain
    Chain []Step // each a StepRef (or StepCall when Level B)

    // StepBranch
    BranchValue string // value-ref name
    BranchIsIt  bool   // Level B
    Cases       []Case

    // StepLoop / StepRepeat
    Cond     *Cond
    Max      *int   // nil if absent
    HasMax   bool
    Body     []Step
    DoWhile  bool // true for repeat (condition after body)

    // StepParallel
    PFork   []Step // branch steps
    Gather  *Step  // gather atom (a StepRef with optional alias); nil if absent
    Each    *Each  // Level B

    // StepCall (Level B)
    CallArgs  []ast.Value
    CallBlock []Step
}

type Case struct{ Values []string; Step Step; Pos lexer.Position }
type Cond struct{ Value string; IsIt bool; Op string; Enum string; Pos lexer.Position } // Op == "==" | "!="
type Each struct{ Item, As string; Pos lexer.Position }
```

> `Max` conversion: the AST holds `*float64`. M2 converts to `int`; a non-integer
> (`max 3.5`) or negative value emits `AF000`-class error here (reuse `AF000` or a new
> `AF112`; **decision: use `AF000` "max must be a positive integer"** to avoid a new
> code, since it is a literal/syntax-shaped error). Document the choice in code.

## 4. Resolver entry point (`internal/sema/resolve.go`)

```go
func Resolve(root *ast.AST, srcDir string) (*model.Program, diag.Diagnostics)
```

`srcDir` is the directory of the `.af` file (for prompt-path resolution; §6). Pipeline
order inside `Resolve`:

1. **First pass — declare.** Walk `root.Decls` in order; build symbol-table entries for
   every `use`/`type`/`agent`/`gate`/`flow` with names only (and `Raw` field bags). Emit
   `AF200`-precursor? No — duplicate-name detection is M4 (`AF200`). But M2 still needs a
   deterministic map; on duplicate, **last wins in the map but record both in `Order`**,
   and leave the duplicate diagnostic to M4. (Rationale: keep M2 resolution total; M4 is
   the single place that reports duplicates.)
2. **Second pass — interpret fields** per kind (§5), populating typed fields and `Raw`.
3. **Model resolution** for every agent (§5.2).
4. **Prompt resolution** for every agent (§6).
5. **Resolve step trees** for every flow (AST steps → `model.Step`), detecting Level B
   markers and emitting `AF150` (§7).
6. **Entry-flow accounting** (§5.6).
7. Register **implicit terminals** `done` and `fail` (§5.7).

Always run all passes (do not early-return on first error) so the user sees a full
diagnostic set. Resolution must be **total**: never panic on malformed-but-parsed input.

## 5. Field interpretation (`internal/sema/fields.go`)

Table-driven. For each kind, a map from known key → handler. Unknown keys: append an
`AF1xx` **warning** (`AF120 unknown field`, severity Warning) and store the raw value in
`Raw[key]`. (`AF120` is a new code in the `AF1xx` resolve range; record it in the
catalog.)

### 5.1 `use` (capability)

| Key | Handling |
|-----|----------|
| `kind` | string; sets `Capability.Kind`. |
| `models` | list of idents → `[]string`; required when `kind: model-provider`. |
| `tools` | list of idents → `[]string` (for mcp). |
| `transport` | string. |
| `command` | string. |
| `args` | list of strings. |

Structural checks (M2-local; these are *resolution* errors, not M4 validation):

- `kind: mcp` without `command:` → `AF130` "mcp capability requires command". (New code.)
- `kind: model-provider` without `models:` → `AF131` "model-provider requires models list".

> Decision: capability structural errors live in the `AF13x` band (resolve-time
> structural), distinct from `AF2xx` (post-inline validation). Add `AF130`, `AF131` to
> the catalog. Alternative considered: fold into `AF2xx`; rejected because these are
> single-declaration shape errors knowable before inlining.

### 5.2 Model resolution (spec §8)

For each agent's `model:` value:

1. If qualified (`provider.alias`): the provider must exist with `kind: model-provider`
   and list `alias` in `models:`. Else `AF110` (qualified variant: "provider X has no
   model alias Y" / "no provider X").
2. If unqualified (`alias`): scan all `model-provider` capabilities whose `models:`
   contains `alias`.
   - Exactly one → bind `ModelProvider`, `ResolvedAlias`.
   - Zero → `AF110` "unknown model alias 'alias' (no provider lists it)".
   - More than one → `AF110` "ambiguous model alias 'alias'; candidates: provider1, provider2 (qualify as provider.alias)".
3. Missing `model:` on an agent → `AF132` "agent requires a model" (new resolve code).
   (Note: spec §7.3 lists `model` as interpreted; an agent with no model cannot bind.)

`AF110` lists candidate providers sorted alphabetically for deterministic messages.

### 5.3 `agent` fields (spec §7.3)

| Key | Handling |
|-----|----------|
| `model` | required; feeds §5.2. |
| `in` | type name (opaque nominal allowed; no existence check here — M4 `AF202`). |
| `out` | enum type name or `text`; existence/membership is M4. |
| `tools` | list of qual-names → `[]ToolRef` (split on first `.`). Existence is M4 `AF206`. |
| `permissions` | string. |
| `retry` | number → int (output-protocol parse retries); default 0; negative → `AF000`. |
| `prompt` | string; feeds §6 (overloaded inline-or-path). |
| `prompt-file` | string; feeds §6 (always path). |
| `description` | string. |

`prompt` + `prompt-file` mutual exclusion is detected in M4 (`AF211`), but M2 still must
**resolve** whichever path(s) are present so render never touches disk. When both are
present, M2 resolves `prompt-file` (and records the conflict marker so M4 can emit
`AF211`); document this precedence in code.

### 5.4 `type` (enum)

`EnumType{Name, Values}`. No structural checks beyond ≥1 value (already enforced by
grammar). Duplicate enum members within one type → `AF133` warning (new) — optional;
**decision: include as a warning** since it is cheap and helpful, store dedup'd values.

### 5.5 `gate` fields (spec §7.4.1)

| Key | Handling |
|-----|----------|
| `run` | string (script path). |
| `on-fail` | one of `halt`/`retry`/`goto`/`enter-loop`. **`bounce-back` → `AF134` error** ("bounce-back is not valid; use retry or goto with on-fail-target"). Unknown value → `AF134`. Absent → default `FailHalt`. |
| `on-fail-target` | control label string; **required** when `on-fail` is `retry` or `goto` (else `AF135` "on-fail: retry requires on-fail-target"). For `halt`/`enter-loop`, a present target is a warning `AF136`. |
| `behavior` | `blocking` or `advisory`; default **`advisory`** when absent (spec §7.4.1: blocking requires explicit `behavior: blocking` + host hook support). |
| `retry` | number → int script-exit retries; default 0. |

`enter-loop` validity (must be inside a loop) is a **structural-position** check — it
needs the flow context, so defer it to M4 (it depends on where the gate is *used*, and a
gate may be referenced in multiple flows). Record only the action here.

> Add `AF134`, `AF135`, `AF136` to the catalog (gate on-fail resolve errors/warnings).

### 5.6 `flow` header (spec §4.4, §7.5)

| Key | Handling |
|-----|----------|
| `on` | trigger string (only meaningful on entry flow; warn `AF137` if present on non-entry). |
| `in` | input type (opaque nominal allowed). |
| `out` | output type; **optional**. Set `OutExplicit = (present)`. |
| `return` | value label; **optional**. Set `ReturnExplicit = (present)`. |

Do **not** default `Return`/`Out` here — that requires the resolved/inlined graph
(terminal producer), which is M3. M2 only records the explicit/absent markers.

### 5.7 Entry flow + terminals

- Count flows with `Entry == true`. Zero → `AF138` "no entry flow". More than one →
  `AF139` "multiple entry flows: a, b" (list names). Exactly one → set `EntryFlow`.
  (These are program-level resolve errors; add `AF138`/`AF139`.)
- Register `done` and `fail` as implicit terminals — represent as reserved names that
  `Step.Ref` resolution (M4 `AF201`) treats as valid leaves. Store a small set
  `model.Terminals = {"done","fail"}` (a package var/func), not map entries, to avoid
  colliding with the agent/gate/flow namespaces.

## 6. Prompt source resolution (`internal/sema/prompt.go`, spec §7.3.1)

For each agent, determine the prompt and resolve any file to text **at compile time** so
render (M6) never reads disk.

Detection:

- If `prompt-file:` present → it is **always** a path.
- Else if `prompt:` present and its value ends with `.md` (case-insensitive) → it is a
  **path**.
- Else if `prompt:` present → **inline text**; `Agent.Prompt = value`, `PromptFromFile = false`.
- If neither present → `Agent.Prompt = ""` (allowed; M4/M6 may warn, but not an error).

Path handling (when a path is detected):

1. Must be **relative** (reject absolute, e.g. leading `/` or a Windows drive) — record a
   marker; the actual `AF211` diagnostic is emitted in **M4** (spec assigns `AF211` to
   M4). M2's job: attempt resolution and attach a `PromptError` marker to the agent for
   M4 to convert. **Decision:** to keep render disk-free and M4 simple, M2 *reads the
   file when safe* and stores either resolved text or a structured `PromptResolution`
   error reason (`absolute`, `escapes-tree`, `missing`, `unreadable`, `not-utf8`,
   `conflict`). M4 maps reasons → `AF211` messages. Store on `Agent`:

```go
type PromptResolution struct {
    Path    string // cleaned relative path attempted
    OK      bool
    Reason  string // "" if OK; else one of the reasons above
}
```

2. Resolve against `srcDir`: `clean = filepath.Clean(filepath.Join(srcDir, path))`. Reject
   if `clean` escapes `srcDir` (compute `rel, _ := filepath.Rel(srcDir, clean)`; reject if
   `rel == ".." || strings.HasPrefix(rel, ".."+sep)`). Reason `escapes-tree`.
3. Read file. Missing → `missing`. Read error → `unreadable`. `!utf8.Valid(bytes)` →
   `not-utf8`. Else `Agent.Prompt = string(bytes)`, `PromptFromFile = true`,
   `PromptPath = path`, `Resolution.OK = true`.
4. `${NAME}` inside prompt text is **left unexpanded** (no processing).

> The output-protocol suffix (spec §9) is **not** appended here — that is render (M6),
> applied after the final prompt text is known. M2 stores only the base prompt.

## 7. Level B rejection (`AF150`, spec §3.2)

While resolving step trees and headers, emit `AF150 "<construct> unsupported in v0.1
(Level B)"` for:

- Flow `Params` non-empty.
- A step that is a call (`StepCall`: `Atom.Args != nil` or `Atom.Block != nil`).
- `parallel each …` (`Parallel.Each != nil`).
- An explicit `it` value-ref (`ValueRef.It` in a branch head or a condition).
- `use … as …` alias form (`Use.Alias != nil`).
- Edge attributes (`Chain.Attr != nil`) — `[when …, max …]` is Level B in v0.1 step
  position (only `loop`/`repeat` carry conditions in Level A).

Emit one diagnostic per occurrence with its position. Continue resolving (do not abort);
downstream passes simply will not see executable semantics for those nodes (M4 may add
follow-on diagnostics, which is acceptable — the `AF150` is the primary signal).

## 11. Risks & notes

- **Diagnostic code sprawl.** M2 introduces several `AF11x`–`AF13x` codes not in the
  original plan table. Update [plans/README.md](../plans/README.md) catalog and the M4
  doc in the same change so the catalog stays authoritative. (Tracked in the index
  update task.)
- **Prompt read here vs M4.** Reading in M2 (and only *reporting* in M4) keeps render
  disk-free and centralizes `AF211` text in M4. If this split feels awkward during
  implementation, the alternative (read+report both in M4, pass `srcDir` through M3/M5)
  is acceptable — but then M5/M6 must carry `srcDir`, which we explicitly avoid.
- **`done`/`fail` namespace.** Keep terminals out of the decl maps so a user-defined
  agent named `done` is still resolvable (M4 decides precedence; default: user decl wins,
  warn). Note this for M4.
- **Map nondeterminism.** Never iterate a Go map for output; always go through `Order`
  or a sorted key list. This is the #1 source of flaky snapshots.
